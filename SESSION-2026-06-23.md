# Session notes — 2026-06-23

## Bug fixes

- **Transaction tab highlight stuck on "Pengeluaran"** (`7642a01`) — clicking
  Pemasukan/Transfer re-rendered the form body but never moved the `active`
  class to the clicked tab button. Fixed in `assets/app.js`.
- **Stale `monthKey`** (`3a0ef96`) — "Bulan Ini" was computed once at page
  load, so it could go stale if the tab stayed open across a month
  rollover. Moved into `render()` so it's recomputed every render.

## New: admin/biaya + cashback on transactions (`58576eb`)

Expense, income, and transfer forms now have optional "Admin/Biaya" and
"Cashback" fields. On save, a non-zero admin fee spins off an extra expense
transaction (same account as the main transaction); a non-zero cashback
spins off an extra income transaction into a chosen account. Added "Admin /
Biaya" to `KATEGORI` and "Cashback" to `JENIS_MASUK`.

## Codebase audit + cleanup (`3a0ef96`)

Ran a ponytail-style over-engineering audit. Two no-op refactors applied:

- `utf8b64`/`b64utf8` (GitHub sync's base64 transport) reused the existing
  `enc`/`dec`/`b64`/`unb64` helpers instead of the deprecated
  `escape`/`unescape` trick.
- Extracted `numIn(id)` to replace four copies of
  `parseInt(el(id).value.replace(/\D/g,""),10)`.

No behavior or data-format changes — verified safe for the existing
encrypted vault (`localStorage` key `saku.vault.v1`) and GitHub sync copy.

## New features (`313d949`)

1. **Category budgets** — "Atur anggaran" sets a monthly limit per
   category (`S.budgets`); the category bar in "Bulan Ini" turns red and
   shows `spent / limit` when exceeded.
2. **Transaction search + date range** — search box and from/to date
   inputs above the transaction list.
3. **Net worth trend chart** — sparkline in the hero card built from the
   last 6 month-end net-worth snapshots, via `calc(cutoff)`.
4. **CSV export** — "Unduh CSV transaksi" in Settings, alongside the
   existing JSON backup.
5. **Recurring transactions** — "Transaksi Berulang" in Settings: add a
   monthly expense/income template (day-of-month); `processRecurring()`
   auto-creates due occurrences on each unlock, catching up missed months.

Old vaults without `budgets`/`recurring` fields are normalized
(`normalizeState()`) on unlock, import, and cloud pull/connect — no
migration needed.

All five features were driven end-to-end in a real headless-Chromium
session (create vault → set budget → trigger over-budget bar → search →
download CSV → add recurring template) with zero console errors before
committing.

## Where data lives

- `localStorage` key `saku.vault.v1` — AES-256-GCM encrypted blob of the
  whole app state, key derived from the user's password via PBKDF2.
- Optionally GitHub — same encrypted blob pushed as `vault.json` to a
  private repo (`config.js` → `sync.owner`/`sync.repo`), if Cloud Sync is
  turned on.
